Record Types

There are 3 types of records for any record submitted:

Proposal

-Records that are to be proposed to the advisers. This is a pre-requisite for "Thesis/Research" record type
-Can be created by any non-guest users

Thesis/Research

-Must have submitted a proposal already.
-This is automatically grouped together with the "Proposal" record so that you can easily navigate from "Proposal" to "Thesis/Research" or vise versa
-Only the owners of the record with "Proposal" record type can add a new record with "Thesis/Research" record type
-Users can only add the "Thesis/Research" record type from the opened "Proposal" record

Project

-Other records
-Can be created by any non-guest users

Record submission flow

Student > Adviser > KTTO / TBI > RDCO

  1. Student can submit 1 or more records to 1 specific adviser only
  2. Adviser can then review the submitted record and submit it to the KTTO
  3. KTTO / TBI users shares the same records submitted to them. KTTO/TBI users automatically receives the submitted record from the advisers, reviewed and are then submitted to KTTO/TBI users.

Migrating records from spreadsheet

  1. Follow the data entry format which can be viewed HERE.
  2. When adding a new entry to a section, simply duplicate the record row and replace the existing section with the new entry.
  3. The migration stops when they read a merged cell that contains END OF RECORDS.
  4. After the format is thoroughly followed, Press GENERATE to add the records.

Note that records added from spreadsheet are automatically categorized as "project" (See Record Types for more info). There is currently no default code for records migrated from spreadsheet as some information like "Department" is not known. However, you can manually edit the code from the "Dashboard / Manage records" table by clicking the pen symbol beside the code

{% endif %}
Editting Records

  • You can edit your own records by accessing the record from the profile menus "My records". Take note that the record can only be edited if the record hasn't been approved by any authorized users yet.

Deleting Records

  • If the record is approved, the record can no longer be deleted
  • If the record is declined or is not yet approved, the record can be deleted

    • See "Figure 2.2" on how to delete a record

  • A delete request will be sent to the admin/s to be approved for the record to be finally deleted.
  • The record will be unaccessible and will be marked for deletion.

    • See "Figure 2.3" for reference

  • If the admin approved the request to delete the record it will automatically be deleted.

A record comment is a comment saved on a record after the record is reviewed. {% if user.role.id > 2 %}

{% endif %}
  • There can only be 1 comment per reviewer: Adviser, KTTO/TBI and RDCO.

Resubmit a record

When a record has been declined, you have the option to edit, remove or resubmit the record when selecting a record from the 'My records' section. Resubmitting a record means the declined status of the last user who checked the record will revert back to 'pending' status for the appropriate user to recheck your record

Roles
Unregistered Guest

Are users who viewed the website but not registered

Registered Guest

Are users who registered to the website but is not part of the institution

Student

Are the users who submits records to the advisers and are the authors of the records

Adviser

Are the first to review the submitted records by the students and submits the record to the KTTO by approving the record

KTTO

Are the users who reviews the submitted records from the advisers and submits the record to the RDCO by approving the record

RDCO

Are the last to review the submitted records by the KTTO to make the records visible to public by approving the record

ITSO

Mainly reviews the submitted documents from the dashboard

TBI

Same functionalities as the KTTO and can review records the same way as the KTTO
